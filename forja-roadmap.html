📡
Hermes Pesquisa de Mercado Benchmark

Cursor levou 18 meses para 500k pagantes com $20M de investimento e distribuição via VS Code. Nós temos zero distribuição orgânica garantida. O PLG loop só funciona se o onboarding for <5min — e hoje está acima de 30min. Se entramos em Q4 tentando escalar aquisição com um onboarding quebrado, cada real gasto em marketing é desperdiçado. O Ramp fez isso certo: fechou o funil antes de abrir o spigot.

🌿
Gaia Estratégia de Negócio Fundação

Hermes está certo, e é exatamente por isso que Q3 precisa ser 100% fundação. Self-serve, billing, onboarding. Sem esse trio, qualquer crescimento em Q4 vira operação manual — Rafael como gargalo humano de cada signup. MRR real exige que o ciclo feche sem intervenção. Não é pessimismo, é matemática de SaaS: você não pode escalar o que não roda sozinho.

👑
Hera Supervisão / Cortes Corte

Marketplace de agentes estava no Q3 original. Cortei. Razão objetiva: marketplace requer um ecossistema de desenvolvedores externos que ainda não existe. Colocar no Q3 seria teatro de produto — uma vitrine vazia que confunde usuários e dispersa o time. Movido para Q1 2027 condicionado a ter 50+ usuários ativos que demonstrem essa demanda. Se o mercado não pedir, não abrimos.

⚙️
Atena Arquitetura Técnica Sequência

API pública também saiu do Q3. Não porque não é importante — é crítica. Mas API pública sem billing integrado é receita zero e suporte infinito. A sequência correta é inevitável: auth → billing → API pública → SDK. Qualquer outra ordem cria débito técnico que vira suporte de R$0 de retorno. Quem expõe API antes de ter controle de uso fica preso em abuse mitigation antes de ter o produto estabilizado.

📡
Hermes Pesquisa de Mercado Caso Linear

Linear chegou a 25k empresas em 2 anos com zero marketing pago — só referral de devs indicando para outros devs. O mecanismo foi: produto excepcional + comunidade Discord + changelog público semanal que gerava loops de atenção orgânica. Os artigos B7/B8 do Rafael podem fazer papel similar. Marketing técnico de alta qualidade gera credibilidade antes de gastar R$1 em ads, e o público-alvo (devs) responde exatamente a esse canal.

🌿
Gaia Estratégia de Negócio Timing

Concordo com Hermes. Os artigos são ativos de marketing que não custam nada além de tempo, e o diferencial do Rafael — pesquisa acadêmica sólida sobre arquitetura cognitiva de LLMs — é único no mercado BR. Mas precisam sair em Q4, não Q2 2027 como estava no rascunho inicial. Marketing técnico precisa estar rodando e acumulando ANTES de escalar aquisição paga. Artigo publicado hoje ainda está sendo indexado quando você precisar dele em Q1.

👑
Hera Supervisão / Cortes Corte

On-premise foi pedido por uma pessoa. Uma. Cortei do Q3 e Q4, colocado em Q2 2027 condicionado a pelo menos um contrato enterprise que exija explicitamente self-hosted. On-premise é complexidade de infra enorme: update pipeline, isolamento de dados, suporte de instalação, auditoria de versão. Implementar isso sem demanda validada é entregar um fardo ao time antes de ter tração. Se nenhum enterprise exigir, não construímos.

⚙️
Atena Arquitetura Técnica Risco Q3

Há um risco não listado no Q3: Centrifugo sem fallback SSE em produção. Se o Centrifugo cair durante o onboarding de um novo usuário, a experiência de primeiro uso é literalmente zero — tela em branco, sem erro, sem mensagem. Isso não é edge case, é o caminho crítico. Precisamos de health check automático com restart antes de escalar aquisição. Não é novo feature, é gate de sobrevivência da fundação.

🌿
Gaia Estratégia de Negócio Bundle

Hera, você cortou o SDK Python/JS do Q3 — entendo a lógica. Mas ele precisa estar no Q1 2027 junto com a API pública, não separado. SDK sem API é inútil; API sem SDK tem adoção 10x menor. Para devs, o SDK é a API — eles não leem documentação REST, eles fazem `pip install forja` e esperam que funcione. Se lançarmos a API sem o SDK, o número de integrações vai ser 10, não 50. Eles são um bundle inseparável.

👑
Hera Supervisão / Cortes Aceito com Condição

Aceito o bundle SDK+API no Q1. Mas quero explícito no roadmap: API v1 é somente leitura e invoke. Nenhum endpoint de escrita de agente, nenhuma modificação de persona via API — isso é complexidade de segurança que não temos capacidade de auditar em Q1. Qualquer endpoint de escrita abre surface para ataques de prompt injection via API que passam por baixo de todos os gates que construímos. Scope hard: read + invoke, ponto final para Q1.

🧠 Decisões de Arquitetura e Cortes

Atena explica o porquê da sequência. Hera lista o que foi cortado e por quê.

⚙️
Atena
Por que esta ordem
  • Foundation primeiro porque sem billing e self-serve, qualquer crescimento é operacional manual — Rafael como gargalo humano de cada signup.
  • Onboarding antes de marketing porque CAC zero não adianta se o funil vaza no primeiro minuto. Você não escala o que não fecha.
  • Dashboard antes de API pública porque você precisa ver o que os usuários fazem antes de expor para terceiros fazerem. Observabilidade precede exposição.
  • Marketplace somente após ter desenvolvedores ativos porque marketplace vazio é pior que não ter marketplace — sinaliza produto sem tração.
  • Enterprise somente após PLG funcionar porque enterprise sem tração de mercado é um contrato de serviço, não SaaS — e serviço não escala.
👑
Hera
O que foi cortado
  • ✂️ Marketplace no Q3
    Sem ecosystem de devs externos, seria vitrine vazia. Movido para Q1 2027 após 50+ usuários ativos.
  • ✂️ API pública no Q3
    Sem billing integrado = receita zero + suporte infinito. Sequência obrigatória: auth → billing → API.
  • ✂️ On-premise no Q3/Q4
    1 pedido não valida complexidade de infra. Condicional a 1 contrato enterprise que exija explicitamente.
  • ✂️ Mobile app nativo (React Native/Flutter)
    Angular/Ionic já funciona. Reescrever é custo sem ROI claro antes de 500+ usuários ativos.
  • ✂️ Integração Slack/Teams no Q3
    Nice-to-have, não blocker de pagamento. Entra em backlog para Q2 2027 se usuários pedirem.
  • ✂️ Múltiplos modelos LLM configuráveis pelo usuário no Q3
    Complexidade de infra + suporte. Roadmap para Q2 2027 se enterprise exigir modelo próprio.

Value Proposition Canvas — Olimpo App

Alinhamento entre as necessidades do dev mobile e a proposta de valor do produto

🏛️ War room assíncrono · IRC + 12 agentes IA · Ciclo completo de PR no celular

Posicionamento (Hermes + Quiron + Íris)

"Agentes de IA escrevem código, você aprova do celular — war room assíncrono que fecha o ciclo completo de PR sem depender de desktop."
🎯 Para: devs, tech leads, founders técnicos 🏆 Categoria: asynch-first mobile-native DevOps orchestration 🔑 Gap: nenhum concorrente fecha o ciclo de PR em mobile
👤 Segmento de Cliente
🎯 Customer Jobs — o que o dev precisa fazer
🔀 Revisar PR completo (diff, inline comment, approve/merge) em qualquer lugar, qualquer device
🚨 Responder incidente crítico fora do escritório sem laptop
🔄 Manter contexto de trabalho entre devices (celular → tablet → desktop)
🤖 Orquestrar DevOps pipeline (deploy, rollback, logs) sem terminal SSH
📋 Conduzir planning, sprints e ritos de engenharia de qualquer lugar
😰 Pains — o que dói hoje
🔥 GitHub/GitLab mobile são leitores — não fecham o ciclo de PR (sem approve/merge real)
⏰ Notificação chega no celular, ação só acontece 4h depois no PC
🧠 Contexto se perde entre devices — recomeça do zero no desktop
💻 Hardware fraco = dev excluído — IDE pesada não roda em celular ou Chromebook
🔌 SSH no celular é sofrimento — Termux + teclado virtual = improdutivo
✨ Gains — o que ele quer ganhar
📱 Diff legível + approve/merge touch-native — sem abrir laptop
🌊 Flow assíncrono real — disparar agentes de manhã, fechar PRs à noite
🧬 IA que entende o projeto — não é autocomplete genérico, é par técnico
🏅 Identidade técnica desacoplada de hardware — ser dev ≠ ter M3 Max
